2) ARRANQUE LOCAL (pasos en orden)
-------------------------------------------------------------------------------
  git clone https://github.com/Karasowl/rindomes.git
  cd rindomes
  npm install

  # Conecta el backend Convex (abre el navegador para login la 1ª vez).
  # Elige el proyecto EXISTENTE "rindomes" / deployment de desarrollo.
  # Esto regenera convex/_generated y ESCRIBE solo NEXT_PUBLIC_CONVEX_URL
  # dentro de .env.local automáticamente.
  npx convex dev          # déjalo corriendo en su propia terminal

  # En OTRA terminal, levanta el front:
  npm run dev             # -> http://localhost:3000

  Si "npx convex dev" no autocompleta el .env.local, créalo a mano con la
  plantilla del punto 3.


3) .env.local  (NO está en git — hay que recrearlo)
-------------------------------------------------------------------------------
  El archivo .env.local está ignorado por .gitignore (correcto: el repo es
  público y no debe contener claves). Plantilla mínima para que la app corra:

      CONVEX_DEPLOYMENT=dev:<tu-deployment-dev>
      NEXT_PUBLIC_CONVEX_URL=https://<tu-deployment-dev>.convex.cloud
      NEXT_PUBLIC_CONVEX_SITE_URL=https://<tu-deployment-dev>.convex.site

  - Los valores de <tu-deployment-dev> los autoconfigura `npx convex dev`,
    o los copias del .env.local de tu PC vieja, o del dashboard de Convex
    (Project Settings -> URL & Deploy Key).
  - .env.example (sí está en el repo) documenta TODAS las variables.

  CLAVES SECRETAS / OPCIONALES  (la app FUNCIONA sin ninguna — es manual-first):
  -----------------------------------------------------------------------------
  Recupéralas de tu gestor de contraseñas (Bitwarden), NO las pegues en el repo.

  a) IA de texto del lado Next (BYOK, ruta /api/ai/capture) -> en .env.local:
        OPENAI_API_KEY=...           (sin ella: cae al clasificador local)
        OPENAI_MODEL=gpt-5-mini      (opcional, default razonable)

  b) Variables que viven en el ENTORNO DE CONVEX, no en .env.local.
     Se setean con:   npx convex env set NOMBRE valor
        OPENROUTER_API_KEY=...    (visión de recibos; sin ella -> heurística local)
        OPENROUTER_MODEL=...      (opcional; default google/gemini-2.0-flash-001)
        RESEND_API_KEY=...        (OTP por email; necesita también AUTH_EMAIL_FROM)
        AUTH_EMAIL_FROM="RindoMes <noreply@tudominio.com>"
        RINDOMES_ALLOW_TEST_GRANTS=true   (solo pruebas: otorgar Pro a mano)
     Sin OTP, el login degrada a solo-contraseña (sigue funcionando).

5) DESPLIEGUE (Vercel + Convex) — referencia
-------------------------------------------------------------------------------
  - Proyecto Vercel: "rindomes".
  - vercel.json ya define el build como:
        npx convex deploy --cmd 'npm run build'
    => cada deploy de Vercel TAMBIÉN despliega Convex a producción. No hay
       paso manual de "deploy de Convex" por separado.
  - La CONVEX_DEPLOY_KEY de producción vive en las Environment Variables de
    Vercel (no en tu .env.local). Para desarrollo local NO la necesitas.
  - Deploy manual desde la CLI:  vercel  (o  vercel --prod).
